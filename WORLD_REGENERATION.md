# Player-Controlled World Regeneration

## Goal

Allow players to apply newly unlocked world-generation features near their existing settlements instead of forcing them to travel farther and farther to find newly generated terrain.

Regeneration must always be deliberate. Player buildings and important places are preserved unless the player explicitly chooses otherwise.

## World Revisions

Changing terrain-generation settings creates a pending **world revision**. A revision can contain player-selected changes such as:

* Biome distribution and biome transitions
* Terrain height, mountain ridges, erosion, or water level
* Rivers, lakes, caves, ravines, and underground rooms
* Trees, plants, snow, beaches, and other decoration
* Ore types, vein sizes, and ore distribution
* Structures or other generation features unlocked during gameplay

The player selects which revision features to apply and which chunks may use them before regeneration begins.

## Chunk Selection and Protection

The world map should show chunks in clear categories:

* **Protected:** Never regenerated by the current revision
* **Selected:** Eligible for regeneration with the chosen settings
* **Transition:** Blended between protected and regenerated terrain
* **Unchanged:** Not included in the revision

Chunks containing player edits, containers, machines, important structures, or claimed settlement space should be protected automatically. Players can add or remove manual protection through an in-world structure such as a World Anchor or through the world map.

Protecting a chunk should also offer a small safety margin so buildings, roads, bridges, and farms crossing chunk boundaries are not cut in half.

## Regeneration Modes

Players should be able to choose how deeply selected chunks are changed:

1. **Decoration refresh** — Regenerate trees, plants, snow, and surface details while preserving terrain shape and player edits.
2. **Underground refresh** — Regenerate eligible caves, ores, and underground features while preserving protected areas and the surface.
3. **Terrain reshaping** — Regenerate natural terrain, fluids, biomes, and selected features while preserving protected content.
4. **Full regeneration** — Rebuild the entire chunk from the current world revision. This should require explicit confirmation and normally be limited to chunks without protected content.

Internally, this can use the terrain-generation stage mask so the server reruns only the selected stages instead of always replacing the complete chunk.

## Transition Chunks

Chunks surrounding protected areas should blend old and new generation rather than producing cliffs, cut rivers, or obvious square borders. Surface height, water edges, biome weights, and decoration density can gradually transition across one or more chunk rings.

Protected player blocks always win over generated blocks inside the transition area.

## Player Flow

1. The player unlocks or activates new world-generation features.
2. The game creates a pending world revision.
3. The player chooses the features and regeneration mode.
4. The map previews protected, selected, transition, and unchanged chunks.
5. The player adjusts protection and confirms the revision.
6. The authoritative server regenerates selected chunks and saves the new generation metadata.

The preview should clearly warn about any player-modified content that is not protected.

## Persistence and Multiplayer

World revisions are server-authoritative. The server stores:

* The active terrain configuration and revision identifier
* Each chunk’s generation configuration signature and completed stages
* Automatic and manual protection state
* Player edits that must survive regeneration

Clients only request revisions, edit the selection, and display previews. The server validates the request, performs regeneration, persists the result, and sends updated chunks to all connected players.

## Design Rule

**Preservation is automatic; regeneration is chosen.**

New terrain features should be accessible near established players without making their work feel temporary or unsafe.
